## 1. 제품 개요

카테고리 분류와 종료일 기반 상태 구분을 지원하는 개인 할일 관리 웹 애플리케이션.  
기존 앱의 한계(카테고리 미지원, 기한 표현 부재)를 해소하여 개인 워크플로우에 최적화된 도구를 제공한다.

---

## 2. 사용자 페르소나

| 구분 | 페르소나 A | 페르소나 B |
|---|---|---|
| **이름** | 김철수 | 박소영 |
| **연령/역할** | 20대 직장인 | 30대 가정주부 |
| **주요 니즈** | 업무·개인 할일을 카테고리로 구분하고 마감일을 한눈에 파악 | 육아·가사·개인 일정을 분리 관리하고 완료/미완료를 명확히 구분 |
| **사용 환경** | 출퇴근 중 모바일, 업무 중 데스크톱 | 주로 모바일 |
| **핵심 불만** | 기존 앱은 카테고리 없이 할일이 뒤섞임 | 기한이 지난 할일을 별도로 확인하기 어려움 |

## 3. 범위 (Scope)

### 3.1 Phase 1 — MVP (2026-04-28 ~ 2026-04-30)

| 기능 영역 | 포함 기능 |
|---|---|
| **사용자 정보 관리** | 회원가입, 로그인, 로그아웃, 내 정보 조회 |
| **카테고리 관리** | 카테고리 생성·수정·삭제, 목록 조회 |
| **할일 관리** | 할일 생성·수정·삭제, 완료 처리, 목록 조회(카테고리 필터·상태 필터) |

### 3.2 Out of Scope (Phase 1에서 제외)

- 다크모드
- 다국어(i18n) 지원
- 알림 / 푸시
- 반복 일정
- 팀 공유 / 협업
- 소셜 로그인

### 3.3 Phase 2 (일정 미정)

- 다크모드 지원
- 다국어 지원

## 4. 기능 요구사항

### 4.1 사용자 정보 관리

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-U01 | 이메일과 비밀번호로 회원가입할 수 있다 | Must |
| FR-U02 | 이메일과 비밀번호로 로그인할 수 있다 | Must |
| FR-U03 | 로그아웃할 수 있다 | Must |
| FR-U04 | 내 이메일 정보를 조회할 수 있다 | Must |

### 4.2 카테고리 관리

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-C01 | 카테고리를 생성할 수 있다 (이름 필수, 최대 20자, 사용자 내 중복 불가) | Must |
| FR-C02 | 카테고리 이름을 수정할 수 있다 | Must |
| FR-C03 | 카테고리를 삭제할 수 있다 | Must |
| FR-C04 | 카테고리 삭제 시 소속 할일은 미분류 처리되며 삭제되지 않는다 | Must |
| FR-C05 | 내 카테고리 목록을 조회할 수 있다 | Must |

### 4.3 할일 관리

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-T01 | 할일을 생성할 수 있다 (제목 필수 최대 100자, 설명 선택 최대 500자, 카테고리 선택, 종료일 선택) | Must |
| FR-T02 | 할일을 수정할 수 있다 | Must |
| FR-T03 | 할일을 삭제할 수 있다 (영구 삭제, 삭제 전 확인 필요) | Must |
| FR-T04 | 할일을 완료 처리하거나 완료를 취소할 수 있다 | Must |
| FR-T05 | 전체 할일 목록을 조회할 수 있다 | Must |
| FR-T06 | 카테고리별로 할일을 필터링할 수 있다 | Must |
| FR-T07 | 상태(미완료·완료·기한 초과)별로 할일을 필터링할 수 있다 | Must |
| FR-T08 | 종료일이 현재 시각을 초과하고 미완료인 할일은 자동으로 기한 초과로 표시된다 | Must |

## 5. 비기능 요구사항

| ID | 항목 | 요구사항 |
|---|---|---|
| NFR-01 | **인증 보안** | JWT (HS-512) 기반 인증. 비인증 사용자는 회원가입·로그인 화면 외 접근 불가 |
| NFR-02 | **데이터 격리** | 사용자는 본인 소유 데이터만 CRUD 가능 |
| NFR-03 | **플랫폼** | 반응형 웹 (모바일·데스크톱 대응) |
| NFR-04 | **응답 속도** | 주요 API 응답 1초 이내 (단일 사용자 기준) |

## 7. 제약사항 및 가정

- TypeScript는 프론트엔드·백엔드 모두 사용하지 않는다.
- ORM 라이브러리(Prisma 등)는 사용하지 않으며, 원시 SQL을 pg 라이브러리로 직접 실행한다.
- Phase 1 기간은 2026-04-28 ~ 2026-04-30 (3일)으로, MVP 기능에만 집중한다.
- 다크모드와 다국어 지원은 Phase 2에서 구현하며, Phase 1 코드 작성 시 확장 가능한 구조를 유지한다.
