### SCN-01. 신규 사용자 회원가입 및 첫 로그인

**페르소나:** 김철수 (20대 직장인)

**상황(Context)**
입사 후 업무와 개인 할일이 뒤섞이는 문제를 겪던 김철수는 카테고리를 지원하는 할일 관리 앱을 처음 접한다. 앱을 사용하기 위해 계정을 생성하고 로그인한다.

**목표(Goal)**
이메일과 비밀번호로 계정을 생성하고 서비스에 접속한다.

**연결 기능 요구사항:** FR-U01, FR-U02

**단계별 행동 흐름 (Happy Path)**

| 단계 | 행동 | 시스템 반응 |
|---|---|---|
| 1 | 앱에 접속한다 | 회원가입 / 로그인 화면이 표시된다 |
| 2 | "회원가입" 버튼을 누른다 | 이메일·비밀번호 입력 폼이 표시된다 |
| 3 | 이메일(`chulsoo@example.com`)과 비밀번호(`Work1234`)를 입력하고 "가입하기"를 누른다 | 계정이 생성되고 로그인 화면으로 이동한다 |
| 4 | 동일 이메일·비밀번호를 입력하고 "로그인"을 누른다 | JWT 토큰이 발급되고 할일 목록(빈 상태)으로 이동한다 |

**예외/엣지 케이스**

| 케이스 | 상황 | 시스템 반응 |
|---|---|---|
| E-01 | 이미 가입된 이메일로 회원가입 시도 | "이미 사용 중인 이메일입니다" 오류 메시지 표시, 가입 불가 |
| E-02 | 비밀번호가 8자 미만이거나 영문/숫자 미혼합 | 비밀번호 형식 안내 메시지 표시, 가입 불가 |
| E-03 | 잘못된 비밀번호로 로그인 시도 | "이메일 또는 비밀번호가 올바르지 않습니다" 표시, 로그인 불가 |
| E-04 | 이메일 형식이 올바르지 않음 | 이메일 형식 오류 메시지 표시 |

### SCN-02. 업무 카테고리 생성 및 할일 등록

**페르소나:** 김철수 (20대 직장인)

**상황(Context)**
로그인 직후 빈 할일 목록을 확인한 김철수는 업무 관련 할일을 "업무", 개인 일정을 "개인"으로 분리 관리하려 한다. 먼저 카테고리를 만들고 이번 주 마감인 기획서 작성 할일을 등록한다.

**목표(Goal)**
"업무" 카테고리를 생성하고, 종료일이 있는 할일을 해당 카테고리에 등록한다.

**연결 기능 요구사항:** FR-C01, FR-T01

**단계별 행동 흐름 (Happy Path)**

| 단계 | 행동 | 시스템 반응 |
|---|---|---|
| 1 | 카테고리 관리 메뉴로 이동한다 | 카테고리 목록(빈 상태)이 표시된다 |
| 2 | "카테고리 추가" 버튼을 누르고 이름 "업무"를 입력 후 저장한다 | "업무" 카테고리가 목록에 추가된다 |
| 3 | 같은 방식으로 "개인" 카테고리를 추가한다 | "개인" 카테고리가 목록에 추가된다 |
| 4 | 할일 목록으로 이동 후 "할일 추가" 버튼을 누른다 | 할일 생성 폼이 표시된다 (제목, 설명, 카테고리, 종료일 필드) |
| 5 | 제목 "Q2 기획서 작성", 카테고리 "업무", 종료일 "2026-04-30 18:00"를 입력하고 저장한다 | 할일이 등록되고 목록에 "미완료" 상태로 표시된다 |

**예외/엣지 케이스**

| 케이스 | 상황 | 시스템 반응 |
|---|---|---|
| E-01 | 동일 이름의 카테고리를 중복 생성 시도 | "이미 존재하는 카테고리 이름입니다" 오류 표시, 생성 불가 |
| E-02 | 카테고리 이름 20자 초과 입력 | 입력 제한(20자) 또는 오류 메시지 표시 |
| E-03 | 할일 제목 없이 저장 시도 | "제목은 필수 항목입니다" 오류 표시, 저장 불가 |
| E-04 | 할일 제목 100자 초과 입력 | 입력 제한(100자) 또는 오류 메시지 표시 |
| E-05 | 카테고리 미선택으로 할일 저장 | 카테고리가 미분류(null)로 저장되고 목록에 정상 표시 |

### SCN-03. 카테고리 필터로 업무 할일 집중 조회

**페르소나:** 김철수 (20대 직장인)

**상황(Context)**
여러 카테고리에 걸쳐 할일이 쌓인 상황에서, 오늘 집중해야 할 업무 할일만 빠르게 확인해야 한다. 출퇴근 중 모바일로 "업무" 카테고리만 필터링하여 우선순위를 파악한다.

**목표(Goal)**
카테고리 필터를 사용하여 "업무" 카테고리의 할일만 조회한다.

**연결 기능 요구사항:** FR-T05, FR-T06

**단계별 행동 흐름 (Happy Path)**

| 단계 | 행동 | 시스템 반응 |
|---|---|---|
| 1 | 할일 목록 화면으로 이동한다 | 전체 할일 목록(업무·개인·미분류 혼합)이 표시된다 |
| 2 | 카테고리 필터에서 "업무"를 선택한다 | "업무" 카테고리에 속한 할일만 필터링되어 표시된다 |
| 3 | 목록에서 오늘 마감인 할일을 확인한다 | 종료일·상태 정보가 함께 표시되어 우선순위를 파악한다 |
| 4 | 필터를 "전체"로 초기화한다 | 모든 카테고리의 할일이 다시 표시된다 |

**예외/엣지 케이스**

| 케이스 | 상황 | 시스템 반응 |
|---|---|---|
| E-01 | 선택한 카테고리에 할일이 하나도 없음 | 빈 상태 메시지 표시 |
| E-02 | "미분류" 필터를 선택 | 카테고리가 지정되지 않은 할일만 표시 (DR-04) |

### SCN-04. 할일 완료 처리 및 완료 목록 확인

**페르소나:** 김철수 (20대 직장인)

**상황(Context)**
퇴근 전, 오늘 완료한 업무를 처리하고 남은 할일과 완료 내역을 구분해서 확인하고 싶다.

**목표(Goal)**
할일을 완료 처리하고, 상태 필터로 완료된 할일만 조회한다.

**연결 기능 요구사항:** FR-T04, FR-T07

**단계별 행동 흐름 (Happy Path)**

| 단계 | 행동 | 시스템 반응 |
|---|---|---|
| 1 | 할일 목록에서 완료한 항목("Q2 기획서 작성")을 찾는다 | 해당 할일이 "미완료" 상태로 표시된다 |
| 2 | 완료 버튼(체크박스 또는 완료 처리 버튼)을 누른다 | 할일 상태가 "완료"로 변경되어 표시된다 |
| 3 | 상태 필터에서 "완료"를 선택한다 | 완료 처리된 할일만 필터링되어 표시된다 |
| 4 | 상태 필터에서 "미완료"를 선택한다 | 아직 처리하지 않은 할일만 표시된다 |

**예외/엣지 케이스**

| 케이스 | 상황 | 시스템 반응 |
|---|---|---|
| E-01 | 이미 완료된 할일의 완료를 재취소 | 상태가 다시 "미완료"(또는 종료일 기준 "기한 초과")로 변경된다 (DR-03) |
| E-02 | "완료" 필터 선택 시 완료 항목이 없음 | 빈 상태 메시지 표시 |

### SCN-05. 기한 초과 할일 파악 및 마감일 재조정

**페르소나:** 박소영 (30대 가정주부)

**상황(Context)**
박소영은 아이 학교 준비물 구매, 장보기 등 여러 할일을 등록해 두었다. 바쁜 하루를 보내다 보니 일부 할일의 종료일이 지나버렸다. 기한 초과 항목을 한눈에 파악하고 일정을 재조정하고 싶다.

**목표(Goal)**
"기한 초과" 필터로 미처리 항목을 확인하고, 종료일을 수정하여 재계획한다.

**연결 기능 요구사항:** FR-T07, FR-T08, FR-T02

**단계별 행동 흐름 (Happy Path)**

| 단계 | 행동 | 시스템 반응 |
|---|---|---|
| 1 | 할일 목록 화면으로 이동한다 | 전체 할일이 표시되며, 종료일이 지난 미완료 항목은 자동으로 "기한 초과" 상태로 표시된다 (DR-03) |
| 2 | 상태 필터에서 "기한 초과"를 선택한다 | 기한이 초과된 할일만 필터링되어 표시된다 |
| 3 | 기한 초과 항목 중 "학교 준비물 구매"를 선택한다 | 할일 상세 또는 수정 화면이 열린다 |
| 4 | 종료일을 "2026-04-29 12:00"으로 수정하고 저장한다 | 수정된 종료일로 저장되고, 현재 시각 기준 상태가 재판별된다 |
| 5 | 목록으로 돌아와 해당 항목이 "미완료" 상태임을 확인한다 | 수정된 종료일이 현재보다 미래이므로 "미완료"로 표시된다 |

**예외/엣지 케이스**

| 케이스 | 상황 | 시스템 반응 |
|---|---|---|
| E-01 | 수정한 종료일도 현재 시각 이전으로 설정 | 저장 후 여전히 "기한 초과" 상태로 표시된다 (DR-03) |
| E-02 | 종료일 없이 수정 (종료일 제거) | 기한 초과 판별 대상에서 제외되어 "미완료"로 표시된다 (DR-05) |
| E-03 | 기한 초과 필터 시 해당 항목 없음 | 빈 상태 메시지 표시 |
| E-04 | 타인의 할일 수정 시도 (직접 URL 접근) | 접근 불가 처리 (DR-02) |

### SCN-06. 카테고리 삭제 시 할일 미분류 처리 확인

**페르소나:** 박소영 (30대 가정주부)

**상황(Context)**
박소영은 "봄맞이 대청소"용으로 임시 카테고리를 만들어 쓰다가 청소가 완료된 후 해당 카테고리를 삭제하려 한다. 삭제 후에도 카테고리 안의 할일이 사라지지 않는지 확인이 필요하다.

**목표(Goal)**
카테고리를 삭제해도 소속 할일이 미분류 상태로 보존됨을 확인한다.

**연결 기능 요구사항:** FR-C03, FR-C04

**단계별 행동 흐름 (Happy Path)**

| 단계 | 행동 | 시스템 반응 |
|---|---|---|
| 1 | 카테고리 관리 메뉴로 이동한다 | 카테고리 목록이 표시된다 |
| 2 | "봄맞이 대청소" 카테고리의 삭제 버튼을 누른다 | "삭제 시 해당 카테고리의 할일은 미분류로 이동됩니다. 삭제하시겠습니까?" 확인 다이얼로그가 표시된다 (DR-06) |
| 3 | "확인"을 눌러 삭제를 승인한다 | 카테고리가 삭제되고 목록에서 제거된다 |
| 4 | 할일 목록으로 이동하여 "미분류" 필터를 선택한다 | 기존 "봄맞이 대청소" 카테고리에 속하던 할일이 미분류 상태로 표시된다 (DR-04) |

**예외/엣지 케이스**

| 케이스 | 상황 | 시스템 반응 |
|---|---|---|
| E-01 | 확인 다이얼로그에서 "취소"를 선택 | 삭제가 중단되고 카테고리가 유지된다 |
| E-02 | 할일이 없는 카테고리 삭제 | 별도 경고 없이 카테고리만 삭제된다 |
| E-03 | 카테고리 이름 수정 시 20자 초과 입력 | 입력 제한 또는 오류 메시지 표시, 저장 불가 (FR-C02) |

### SCN-07. 할일 삭제 (확인 절차 포함)

**페르소나:** 박소영 (30대 가정주부)

**상황(Context)**
박소영은 더 이상 필요 없는 할일을 정리하려 한다. 실수로 삭제하지 않도록 확인 절차가 있어야 한다.

**목표(Goal)**
확인 절차를 거쳐 할일을 영구 삭제한다.

**연결 기능 요구사항:** FR-T03

**단계별 행동 흐름 (Happy Path)**

| 단계 | 행동 | 시스템 반응 |
|---|---|---|
| 1 | 할일 목록에서 삭제할 항목을 선택한다 | 할일 상세 또는 삭제 버튼이 표시된다 |
| 2 | "삭제" 버튼을 누른다 | "이 할일을 영구 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다." 확인 다이얼로그가 표시된다 (DR-06) |
| 3 | "삭제" 버튼을 눌러 확정한다 | 할일이 영구 삭제되고 목록에서 제거된다 |

**예외/엣지 케이스**

| 케이스 | 상황 | 시스템 반응 |
|---|---|---|
| E-01 | 확인 다이얼로그에서 "취소"를 선택 | 삭제가 중단되고 할일이 유지된다 |
| E-02 | 타인의 할일 삭제 시도 (직접 API 호출) | 접근 불가 처리 (DR-02) |

### SCN-08. 로그아웃 및 비인증 접근 차단 확인

**페르소나:** 김철수 (20대 직장인)

**상황(Context)**
공용 PC에서 작업을 마친 김철수는 계정을 안전하게 로그아웃하려 한다. 로그아웃 후 다른 사람이 내 데이터에 접근하지 못하는지 확인한다.

**목표(Goal)**
로그아웃 후 인증이 필요한 화면에 접근이 차단됨을 확인한다.

**연결 기능 요구사항:** FR-U03

**단계별 행동 흐름 (Happy Path)**

| 단계 | 행동 | 시스템 반응 |
|---|---|---|
| 1 | 내 정보 또는 메뉴에서 "로그아웃" 버튼을 누른다 | JWT 토큰이 무효화되고 로그인 화면으로 이동한다 |
| 2 | 브라우저 뒤로가기 또는 직접 URL 입력으로 할일 목록 접근을 시도한다 | 로그인 화면으로 리다이렉트된다 (DR-01, NFR-01) |

**예외/엣지 케이스**

| 케이스 | 상황 | 시스템 반응 |
|---|---|---|
| E-01 | 세션 토큰이 만료된 상태에서 API 호출 | 401 Unauthorized 응답 후 로그인 화면으로 리다이렉트 |
| E-02 | 로그인 없이 직접 보호된 URL 접근 | 로그인 화면으로 리다이렉트 (NFR-01) |
