## 1. 프로젝트 개요

### 목적
개인이 할일을 체계적으로 관리할 수 있도록 카테고리 분류와 종료일 기반의 상태 구분을 지원하는 웹 애플리케이션을 개발한다.

### 배경
- 기존 일정·할일 관리 앱은 카테고리 분류나 종료시간 표현이 미흡하여 개인 워크플로우에 맞는 도구가 부재함.
- 사용자가 원하는 카테고리를 자유롭게 정의하고, 할일의 상태(완료·기한 초과 등)를 명확히 구분할 수 있는 애플리케이션이 필요함.

## 2. 핵심 도메인 개념

| 엔티티 | 설명 | 속성 및 제약 조건 |
|---|---|---|
| **User (사용자)** | 애플리케이션을 사용하는 계정 주체. | - 이메일(ID): 필수, 중복 불가<br>- 비밀번호: 8자 이상, 영문/숫자 혼합 |
| **Todo (할일)** | 사용자가 등록하는 개별 할일 항목. | - 제목: 필수, 최대 100자<br>- 설명: 선택, 최대 500자<br>- 종료일: 선택, 일시 포함 |
| **Category (카테고리)** | 할일을 분류하는 사용자 정의 그룹. | - 이름: 필수, 최대 20자, 사용자 내 중복 불가 |
| **Status (상태)** | 할일의 현재 상태. | - `미완료`, `완료`, `기한 초과` |

## 3. 도메인 규칙 (Domain Rules)

| ID | 규칙명 | 세부 내용 |
|---|---|---|
| **DR-01** | **인증 필수** | 모든 기능은 로그인한 사용자만 사용할 수 있다. 비인증 사용자는 회원가입·로그인 화면 외에 접근할 수 없다. |
| **DR-02** | **데이터 격리** | 사용자는 자신이 생성한 할일과 카테고리만 조회·수정·삭제할 수 있다. (소유권 기반 접근 제어) |
| **DR-03** | **상태 자동 판별** | 종료일이 설정된 할일 중 `완료`되지 않은 항목은 **현재 시간 > 종료일**인 경우 자동으로 `기한 초과` 상태가 된다. 판별은 목록 조회 시점 또는 매분 단위 스케줄러에 의해 갱신된다. |
| **DR-04** | **카테고리 자유 정의** | 시스템 기본값 없이 사용자가 직접 생성한다. 카테고리가 삭제되면 해당 카테고리에 속한 할일은 `미분류(null)` 상태가 되며, 할일 자체는 삭제되지 않는다. |
| **DR-05** | **종료일 가변성** | 종료일은 필수가 아니며, 설정하지 않은 할일은 기한 초과 판별 대상에서 제외되어 상시 `미완료` 상태를 유지한다. |
| **DR-06** | **데이터 삭제 정책** | 모든 데이터(할일, 카테고리) 삭제는 **영구 삭제(Hard Delete)**를 원칙으로 한다. 단, 삭제 전 사용자 확인 절차를 반드시 거친다. |

## 5. 용어 사전 (Ubiquitous Language)

| 용어 | 설명 |
|---|---|
| **할일 (Todo)** | 사용자가 수행해야 할 단위 작업 항목 |
| **카테고리 (Category)** | 할일을 논리적으로 묶는 사용자 정의 분류 단위 |
| **종료일 (Due Date)** | 할일을 완료해야 하는 목표 일시 |
| **완료 (Completed)** | 사용자가 명시적으로 완료 처리한 할일 상태 |
| **기한 초과 (Overdue)** | 종료일이 현재 일시를 지났으나 아직 완료되지 않은 상태 |
| **미완료 (Pending)** | 종료일이 남아 있거나 종료일 없이 아직 완료되지 않은 상태 |
| **인증 (Authentication)** | 회원가입 및 로그인을 통해 사용자 신원을 확인하는 과정 |
